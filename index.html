Understanding and Detecting Flaky Builds in GitHub Actions

Project Overview

This research project investigates flaky builds in continuous integration (CI) systems, providing comprehensive failure analysis data and predictive models. Our work aims to help developers identify, understand, and mitigate build failures through automated tools and machine learning approaches.

All data, models, and tools are openly available to support reproducible research and practical application in software development workflows.

Failure Analysis Data

This comprehensive dataset presents real-world flaky build failures collected from GitHub Actions CI pipelines across multiple open-source projects. Each entry documents a specific build failure case, including the root cause analysis, error symptoms observed in logs, and both short-term fixes and long-term prevention strategies. The data is organized by failure categories to help developers quickly identify patterns and apply appropriate mitigation approaches.

The table includes:
(1) Failure Category - the high-level classification of the build failure;
(2) Repository - the GitHub repository where the failure occurred;
(3) Run ID - the unique GitHub Actions run identifier;
(4) Root Cause - the underlying technical reason for the failure;
(5) Analysis - detailed explanation connecting observed error symptoms to root causes through log analysis and issue investigation;
(6) Mitigation - actionable short-term fixes and long-term preventive measures;
(7) Screenshots - visual evidence from CI logs illustrating the failure.
